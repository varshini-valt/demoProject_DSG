About this project: This application was built as part of a demonstration project during my probation period, prior to being confirmed as a permanent employee.

  • No live APIs or backend services are connected — the entire application operates on mock data with a graceful fallback mechanism.
  • All error handling, authentication flows (sign-up, sign-in, forgot password), and edge-case scenarios are fully implemented and can be reviewed throughout the codebase.
  • After completing the sign-up flow, returning to sign in with the same credentials will auto-populate the registration form with previously entered data — as long as the page has not been refreshed, since all state is managed in-memory via a shared Angular service.
